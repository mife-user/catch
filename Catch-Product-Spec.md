## 一、产品概述

### 1.1 产品定位

Catch 是一款面向非技术用户的文件整理工具，通过"命令行启动 + 网页界面操作"的混合架构，帮助用户高效管理本地文件。用户只需记住一个命令 `catch`，即可通过直观的网页界面完成文件查找、删除、重命名、移动/复制等操作。

### 1.2 核心价值

- **零学习成本**：无需记忆复杂命令行参数，所有操作通过网页界面完成
- **安全可靠**：多重安全机制保障用户数据安全
- **高效便捷**：利用 Go 并发特性实现快速文件处理

### 1.3 目标用户

- 普通电脑用户（非技术人员）
- 需要定期整理文件的用户
- 对命令行有抵触情绪的用户

### 1.4 竞品差异化

| 维度 | Catch | find/fd | Everything |
|------|-------|---------|------------|
| 目标用户 | 非技术用户 | 技术用户 | 技术用户 |
| 操作方式 | 网页界面 | 命令行 | GUI |
| 学习成本 | 极低 | 高 | 中 |
| 安全机制 | 多重保障 | 无 | 基础 |

### 2.2 启动模块

#### 2.2.1 命令行启动

**功能描述**：用户在终端输入 `catch` 命令，启动整个应用。

**交互流程**：
1. 用户在任意目录下输入 `catch`
2. 系统检测配置文件是否存在，不存在则创建默认配置
3. 系统检测可用端口（范围：3000-3100）
4. 启动 Gin API 服务
5. 自动打开默认浏览器访问 `http://localhost:{端口}`
6. 终端显示服务状态信息

**终端输出示例**：
```
Catch 文件整理工具 v1.0.0
===============================
正在启动服务...
检测到可用端口: 3000
服务已启动: http://localhost:3000
正在打开浏览器...
按 Ctrl+C 停止服务
```

**错误处理**：
- 浏览器打开失败：提示用户手动访问地址
- 端口全部占用：提示用户关闭占用端口的程序
- 配置文件损坏：自动备份并重建配置

#### 2.2.2 端口自动分配

**功能描述**：自动检测并分配可用端口。

**实现逻辑**：
1. 默认端口：3000
2. 扫描范围：3000-3100（共101个端口）
3. 最大尝试次数：100次
4. 找到可用端口后写入配置文件，下次启动优先使用

#### 2.2.3 首次启动引导

**功能描述**：首次启动时显示交互式教程。

**引导内容**：
1. 欢迎页面
2. 功能介绍（简短）
3. 安全密码设置（可选，跳过则永久删除功能不可用）
4. 开始使用

### 2.3 文件查找模块

#### 2.3.1 功能描述

用户通过网页界面设置查找条件，系统返回匹配的文件列表。

#### 2.3.2 查找条件

| 条件 | 类型 | 说明 |
|------|------|------|
| 搜索路径 | 文本输入 | 默认当前目录，支持浏览选择 |
| 文件名 | 文本输入 | 支持精准匹配/模糊匹配 |
| 文件类型 | 下拉选择 | 全部/文档/图片/视频/音频/自定义扩展名 |

1. 用户在左侧导航点击"文件查找"

3. 用户填写条件，点击"开始查找"

6. 用户可勾选卡片进行批量操作

**卡片内容**：
- 文件图标（根据类型显示）
- 文件名
- 文件路径
- 文件大小
- 修改时间
- 复选框（用于批量选择）

**操作按钮**：
- 全选/取消全选
- 反选
- 删除选中
- 移动选中
- 复制选中
- 重命名选中

#### 2.3.5 权限处理

- 无权限访问的文件自动跳过
- 查找完成后显示"跳过的文件"列表
- 用户可查看跳过原因（权限不足/文件被占用等）

### 2.4 文件删除模块

#### 2.4.1 功能描述

用户可选择将文件移至回收站或直接删除，直接删除支持过期自动清理机制。

#### 2.4.2 删除方式

| 方式 | 说明 | 安全等级 |
|------|------|----------|
| 移至回收站 | 调用系统回收站，可恢复 | 高 |
| 直接删除（过期清理） | 移至 `.catch-trash/`，到期自动清理 | 中 |
| 永久删除 | 立即彻底删除，需密码验证 | 低 |

#### 2.4.3 过期自动清理机制

**实现方案**：
- 删除文件移至用户目录下 `.catch-trash/`
- 每个文件记录删除时间
- 每次启动 catch 时检查过期文件并清理
- 默认过期时间：7天（可在设置中修改）

**数据结构**：
```json
{
  "trash": [
    {
      "original_path": "/path/to/file.txt",
      "trash_path": "/user/.catch-trash/20260409_file.txt",
      "deleted_at": "2026-04-09T10:00:00Z",
      "expires_at": "2026-04-16T10:00:00Z"
    }
  ]
}
```

#### 2.4.4 永久删除安全验证

**流程**：
1. 用户选择"永久删除"
2. 系统弹出密码输入框
3. 用户输入安全密码
4. 验证通过后执行删除
5. 显示删除结果

**密码管理**：
- 首次使用时引导设置
- 明文存储于配置文件 `.catch/config.json`
- 支持修改和重置

### 2.5 文件重命名模块

#### 2.5.1 功能描述

用户可对批量文件进行重命名操作。

#### 2.5.2 重命名规则（当前版本）

| 规则 | 说明 | 示例 |
|------|------|------|
| 添加前缀 | 在文件名前添加指定文本 | `photo_001.jpg` → `vacation_photo_001.jpg` |
| 添加后缀 | 在扩展名前添加指定文本 | `report.pdf` → `report_final.pdf` |
| 序号编号 | 按顺序添加编号 | `img.jpg` → `img_001.jpg`, `img_002.jpg` |
| 替换文本 | 替换文件名中的指定文本 | `photo_001.jpg` → `picture_001.jpg` |
| 日期时间戳 | 添加日期时间信息 | `doc.txt` → `doc_20260409.txt` |

#### 2.5.3 交互流程

1. 用户选择要重命名的文件
2. 选择重命名规则
3. 填写规则参数
4. 预览重命名结果
5. 确认执行
6. 显示执行结果
